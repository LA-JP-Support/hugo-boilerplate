# Glossary Title Translation Guide (用語集タイトル翻訳ガイド)

## Overview
This guide provides rules for translating glossary entry titles from English to Japanese. The key challenge is determining when to keep terms in English vs. when to translate them to Japanese.

## Decision Framework

### Rule 1: Keep in English (英語のまま)
Use English when the term is:
- **Widely recognized in English** globally and in Japan
- **More commonly used in English** in professional contexts
- **A proper noun** (product names, company names, protocols)
- **An acronym** that is universally used (API, SEO, CRM, etc.)
- **Technical jargon** where Japanese translation would be confusing

### Rule 2: Translate to Japanese (日本語に翻訳)
Use Japanese when:
- **Standard Japanese translation exists** and is commonly used
- **General audience term** that non-technical people should understand
- **Japanese translation is clearer** than the English term
- **Official Japanese term** is defined by industry standards

### Rule 3: Hybrid (英語 + 日本語説明)
Use both when:
- **English term is common** but Japanese explanation helps understanding
- **New or emerging terms** that need context
- Format: `English（日本語説明）` or `日本語（English）`

## Special Cases

### Case 1: Acronyms with Japanese Explanation
When an acronym is universally used but needs explanation:

```
Title: API（Application Programming Interface）
Description: アプリケーション間でデータをやり取りするための仕組み
```

### Case 2: English Term with Japanese Subtitle
When English is primary but Japanese helps understanding:

```
Title: Chatbot（チャットボット）
Description: 自動的に会話を行うプログラム
```

### Case 3: Japanese Term with English Reference
When Japanese is primary but English reference is helpful:

```
Title: 機械学習（Machine Learning）
Description: データからパターンを学習するAI技術
```

### Case 4: Hybrid Technical Terms
When the term is a mix of English and Japanese:

```
Title: A/Bテスト
Description: 2つのバージョンを比較して効果を測定する手法
```

## Decision Tree

```
Is it a proper noun (product/company name)?
├─ YES → Keep English (e.g., ChatGPT, Google)
└─ NO → Continue

Is it a universal acronym (API, SEO, CRM)?
├─ YES → Keep English
└─ NO → Continue

Is there a standard Japanese translation?
├─ YES → Is it commonly used?
│   ├─ YES → Use Japanese (e.g., 機械学習, 人工知能)
│   └─ NO → Use Katakana (e.g., チャットボット)
└─ NO → Continue

Is it a technical term?
├─ YES → Use Katakana (e.g., トークン, プロンプト)
└─ NO → Use Japanese if possible (e.g., 設定, 統合)
```

## Quality Checklist

When translating glossary titles, verify:

- [ ] Is the term a proper noun? → Keep English
- [ ] Is it a universal acronym? → Keep English
- [ ] Does a standard Japanese translation exist? → Use it
- [ ] Is the Japanese translation commonly used? → Verify usage
- [ ] Would the target audience understand it better in English or Japanese?
- [ ] Is consistency maintained with similar terms?
- [ ] Does the description clarify the term regardless of language choice?

## Common Mistakes to Avoid

### Mistake 1: Over-translating technical terms
```
❌ WRONG: ジーピーティー (GPT)
✅ CORRECT: GPT
```

### Mistake 2: Keeping common terms in English
```
❌ WRONG: Machine Learning
✅ CORRECT: 機械学習
```

### Mistake 3: Inconsistent katakana usage
```
❌ WRONG: チャットボット (in one place), chatbot (in another)
✅ CORRECT: チャットボット (consistently)
```

### Mistake 4: Translating product names
```
❌ WRONG: チャットジーピーティー
✅ CORRECT: ChatGPT
```

### Mistake 5: Using English for common Japanese words
```
❌ WRONG: Configuration
✅ CORRECT: 設定
```
