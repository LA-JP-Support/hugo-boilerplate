## Gumloopとは？

**Gumloop**は、AIを活用したインテリジェントな自動化ワークフロー（「Flow」）を構築するための、モダンなノーコードプラットフォームです。従来の自動化ツールが厳格なルールベースのトリガーとアクションのみに依存しているのに対し、Gumloopは**AI-native**です。つまり、OpenAIのGPTシリーズやAnthropicのClaudeなどの大規模言語モデル（LLM）を、自動化エンジンのあらゆる層に深く統合するよう、ゼロから設計されています。

### 主な機能:

- **自動化ワークフロー:** 複数のアプリやデータソースにまたがるタスクのシーケンスを構築し、自動的またはオンデマンドで実行します。
- **AIエージェント:** 平易な言語で指示を解釈し、実行するワークフローを決定し、ライブコンテキストに基づいてアクションを適応させるエージェントを展開します。
- **ドラッグ&ドロップビルダー:** キャンバス上でモジュール式の「ノード」を接続することで、自動化を視覚的に設計します。
- **高度なデータ処理:** メール、スプレッドシート、CRM、Webページなどから情報を取得、処理、活用します。

> 「Gumloopは、GTM（Go-to-Market）チーム、レベニューオペレーション、ビジネス開発向けに特別に構築されたAI-native自動化プラットフォームです。AI駆動の自動化を構築するためのノーコードソリューションとして位置づけられています。」  
> — [Softailedレビュー](https://softailed.com/blog/gumloop-review)

### 何が特別なのか？

- **Model Context Protocol（MCP）:** AIモデルを外部ツールやデータに自然言語スクリプティングで接続するためのオープンスタンダード。[MCPスクリプティングについて学ぶ](https://www.gumloop.com/blog/a-different-take-on-mcp-introducing-mcp-scripting)
- **ブラウザ自動化:** AI駆動のWebスクレイピングとブラウザタスクのためのChrome拡張機能。
- **カスタム統合:** 平易な英語で新しい自動化を記述すると、AIがバックエンドロジックを構築します。

## Gumloopの仕組み

Gumloopは、**ワークフロー（Flow）**と**エージェント**という2つの主要なプリミティブを通じて動作します。

### 1. ワークフロー（「Flow」）
- **ビジュアル自動化:** グラフィカルビルダーで設計されたステップバイステップのシーケンス。
- **ノード:** 各ステップは「ノード」です。データベースの読み取り、ページのスクレイピング、AI分析の実行、メール送信などのアクションを表します。
- **トリガー:** Flowは、さまざまなイベント（Webhook、スケジュール、アプリ内の新しいデータ）から開始するか、手動で開始できます。

#### 高度な概念:
- **サブフロー:** 複数の親Flowから呼び出せる、再利用可能なモジュール式ミニワークフロー。コード/データの再利用と保守性を促進します。
- **条件ロジックとループ:** if/then分岐、バッチ処理、エラーハンドリングを使用して、堅牢な自動化を実現します。
- **MCPノード:** Model Context Protocolを活用し、自然言語プロンプトを通じて構築されたカスタムノード。互換性のあるツールやデータセットにアクセスできます。

### 2. エージェント

エージェントは、以下を行うAI駆動のオーケストレーターです:
- 複雑な指示を解釈する（「プロジェクトXに関する未読のGmailスレッドをすべて要約し、オペレーションチームに通知する」など）
- どのFlow/サブフローをどの順序で実行するかを自律的に決定する
- 推論、判断、または複数ステップの問題解決を必要とする、コンテキストを考慮したタスクを処理する
- Slack、Chrome、またはAPI/Webhookを介してトリガーできる

> 「Gumloopは、モデルコンテキストプロトコル（MCP）と呼ばれるものを採用しています。これは、AIアプリケーション用のUSB-Cポートのように機能するオープンソース標準です。これにより、AIシステムを外部データソース、ツール、ワークフローに接続する標準化された方法が提供されます。」

## 主な機能と能力

### ビジュアル自動化ビルダー

- **ドラッグ&ドロップUI:** ノードを接続し、ロジックを視覚化し、デバッグするための直感的なキャンバス。
- **100以上の事前構築済みノード:** 統合、データ処理、AIタスク、ループ、分岐のためのすぐに使えるアクション。
- **サブフロー:** 再利用可能なロジックをネストすることで、複雑な自動化を構成します。

### AI駆動の自動化

- **AIノード:** 要約、データ抽出、分類、コンテンツ生成、セマンティック分析のためのステップ。LLMによって駆動されます。
- **プロンプトから作成:** 自然言語から新しいノードとFlowを構築し、AIが基礎となるロジックを生成します。
- **MCPスクリプティング:** Model Context Protocolを介してツールを呼び出す、再利用可能で編集可能なスクリプトを生成します。会話型チャットボットを超えた、決定論的で反復可能な自動化を提供します。  
  [MCPとは？](https://www.gumloop.com/blog/a-different-take-on-mcp-introducing-mcp-scripting)

### アプリ&データ統合

- **50以上のネイティブ統合:** Gmail、Google Sheets、Slack、Salesforce、Notion、Airtable、Apollo、Zendeskなど。
- **Webサイトスクレイピング:** AIを活用した、Webページからデータを抽出するためのノーコードブラウザ拡張機能。
- **カスタムAPI:** Webhook、RESTfulエンドポイント、直接API呼び出しのサポート。

### 高度なワークフローロジック

- **条件分岐:** データとコンテキストに適応するFlowを構築します。
- **ループモード:** 大規模なデータセットやリストをバッチ処理します（例:スプレッドシートのすべての行を要約）。
- **トリガー:** 実行をスケジュールし、外部イベントをリッスンし、手動キックを処理します。

### テンプレート&パブリックFlow

- **テンプレートギャラリー:** マーケティング、営業、サポート、オペレーション向けの60以上の事前構築済み自動化。  
  [テンプレートを閲覧](https://www.gumloop.com/templates)
- **共有可能なワークフロー:** チームメイトやクライアント向けのミニアプリとしてFlowを公開します。

### セキュリティ&コラボレーション

- **チームワークスペース:** Flowと認証情報を安全に共有します。
- **SOC2/GDPR準拠:** エンタープライズグレードのセキュリティ。データはAIモデルのトレーニングに使用されません。
- **ロールベースアクセス:** チームメンバー間で権限を制御します。

### スケーラブルなインフラストラクチャ

- **並行実行:** 数百または数千の自動化を同時に実行します。
- **バックグラウンドワークフロー:** 手動実行だけでなく、イベントやスケジュールによってトリガーされます。

## Gumloopの比較:何が違うのか？

- **AI-Native:** LLMを活用したコンテキスト対応の自動化のために、初日から構築されています。ルールベースのトリガーだけではありません。
- **MCP対応:** Model Context Protocolを使用して、外部ツールやAPIとのシームレスな自然言語統合を実現します。
- **ビジュアル&会話型自動化:** ドラッグ&ドロップビルダーに加えて、Gummieアシスタントを介したプロンプトベースの作成。
- **再利用可能なコンポーネント:** サブフロー、テンプレート、MCPスクリプトで効率を最大化します。
- **柔軟な統合:** 新旧のSaaS、カスタムAPI、さらにはブラウザベースのデータソースにも接続します。
- **決定論的スクリプティング:** MCPスクリプティングにより、反復可能でスケーラブル、編集可能なワークフローが可能になります。エンタープライズグレードの信頼性に不可欠です。

## Gumloop用語集

- **Flow / ワークフロー:** ノードを使用して視覚的に構築された自動化されたステップのシーケンス。
- **ノード:** ワークフロー内のアクションまたはビルディングブロック（例:メールの読み取り、AIの実行、CRMの更新）。
- **エージェント:** ワークフローを実行し、コンテキストを考慮した決定を行うAI駆動のオーケストレーター。
- **サブフロー:** 他のFlow内に埋め込まれた再利用可能なミニワークフロー。
- **テンプレート:** 使用および適応の準備ができた、事前構築済みのカスタマイズ可能な自動化。
- **トリガー:** ワークフローを開始するイベント（メール、時間、Webhook）。
- **ループモード:** リストまたはデータセットをバッチ処理します（例:すべてのスプレッドシート行を要約）。
- **統合:** サードパーティアプリへの接続（例:Gmail、Salesforce、Notion）。
- **クレジット:** 課金の使用単位。各アクションまたはAI呼び出しはクレジットを消費します。
- **MCP（Model Context Protocol）:** 自然言語スクリプトを介してAIを外部データ/ツールに接続するためのオープンスタンダード。[MCPスクリプティングガイド](https://www.gumloop.com/blog/a-different-take-on-mcp-introducing-mcp-scripting)
- **Ask AIノード:** 複数のMCPサーバーに接続し、LLMを介してツール呼び出しをオーケストレーションできる特別なノード。[ドキュメント](https://docs.gumloop.com/nodes/mcp/ask_ai_mcp_support)
- **ネイティブMCPノード:** 反復可能な本番グレードのMCPワークフロー用に事前設定されたノード。

## よくある質問

**Q: コーディング方法を知らなくてもGumloopを使用できますか？**  
A: はい。GumloopのUIとAIアシスタントは、非技術系ユーザー向けに構築されています。

**Q: Gumloopは何を自動化できますか？**

**Q: GumloopはZapierやMakeとどう違いますか？**
