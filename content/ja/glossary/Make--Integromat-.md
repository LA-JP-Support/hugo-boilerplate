## Make (Integromat)とは?
Make(旧Integromat)は、ノーコード自動化プラットフォームであり、小規模企業から大企業まで、従来のプログラミングなしでアプリ、API、データソース間のワークフローを設計、構築、自動化できます。このプラットフォームは、強力なビジュアルシナリオビルダー、リアルタイム自動化、高度なデータ変換機能、先進的なエラー処理を提供し、最も柔軟で深くカスタマイズ可能な自動化ツールの一つとなっています。

Makeは、生産性ツール(Google Workspace、Slack、Notion)、eコマースプラットフォーム(Shopify、WooCommerce)、マーケティングオートメーション(Mailchimp、HubSpot)、データベース(Airtable、MySQL)を含む2,500以上のアプリに接続し、HTTPモジュールを通じて事実上あらゆるREST APIに対応しています。ドラッグ&ドロップインターフェースは、データマッピング、変換、スケジューリング、高度な条件ロジックを備えた線形または高度に分岐したワークフローの設計をサポートし、複雑なビジネスプロセスの自動化を可能にします。

このプラットフォームは、反復的な手作業の排除、複雑なビジネスプロセスのオーケストレーション、異種システムの統合、または高度な自動化ワークフローの構築を求めるすべての人に適しており、すべてコードではなくビジュアル設定を通じて実現できます。

## Makeの仕組み

Makeは「シナリオ」を通じて動作します。シナリオとは、相互接続されたモジュールから構築されたワークフローの視覚的表現です。各シナリオは、トリガー、アクション、制御フローツールを組み合わせて包括的な自動化ロジックを形成します。

### シナリオアーキテクチャ

<strong>トリガー:</strong>接続されたアプリ内のイベントがシナリオの実行を開始します—新着メール、フォーム送信、データベースレコード作成、Webhook呼び出し、またはスケジュールされた時刻。

<strong>モジュール:</strong>特定のアクション(メール送信、スプレッドシート更新、タスク作成、API呼び出し)またはデータ変換(JSON解析、レコードフィルタリング、結果集約)を実行します。

<strong>ロジックと制御フロー:</strong>条件ロジック(If/Else、Switch)、ルーター(分岐)、フィルター(ゲートキーピング)、イテレーター(配列処理)、アグリゲーター(データ結合)を使用して実行フローを制御します。

<strong>エラー処理:</strong>モジュール内またはグローバルエラーハンドラーを通じてフォールバックアクションを定義し、再試行、スキップ、代替パス、または通知を指定します。

<strong>スケジューリング:</strong>スケジュール(X分/時間/日ごと)、リアルタイムWebhook、または手動開始でシナリオを実行します。

### ビジュアルシナリオビルダー

ドラッグ&ドロップビルダーにより、ユーザーはワークフローを視覚的にマッピングし、モジュール間を流れるデータを検査し、詳細なステップレベルの実行ログでデバッグできます。リアルタイムデータプレビューは、テスト実行中にモジュール間を通過する実際の値を表示します。

### 高度な機能

<strong>カスタム関数:</strong>データ変換、計算、またはシナリオ全体で使用できる複雑な操作のための再利用可能なロジックブロックを構築します。

<strong>HTTPモジュール:</strong>カスタム認証、ページネーション、ヘッダー管理、レスポンス処理を備えた高度なAPI呼び出しを実行します。

<strong>データストア:</strong>シナリオ実行間でデータを永続化し、ステートフルな自動化、マルチステッププロセス、メモリを必要とする複雑なワークフローを可能にします。

<strong>Webhook:</strong>外部システムからデータを受信するカスタムエンドポイントを作成し、リアルタイムでシナリオ実行をトリガーします。

## 主な機能

<strong>ビジュアルワークフロービルダー:</strong>シンプルな線形フローと、ネストされた条件を持つ複雑な分岐ロジックの両方をサポートするドラッグ&ドロップインターフェース。

<strong>2,500以上の統合:</strong>主要プラットフォーム用のネイティブモジュールに加え、カスタム認証方法を備えた任意のREST API用のHTTPモジュール。

<strong>高度なロジック:</strong>フィルターがデータフローをゲートキープし、ルーターが条件に基づいて分岐し、イテレーターが配列/リストを処理し、アグリゲーターが複数のデータバンドルを結合します。

<strong>リアルタイムおよびスケジュール自動化:</strong>Webhookによる即時実行、または分単位の粒度でのスケジュール実行。

<strong>データマッピングと変換:</strong>モジュール間のビジュアルデータマッピング、テキスト操作、日付処理、数学演算、JSON/XML解析のための組み込み関数。

<strong>エラー処理とログ記録:</strong>再試行ロジックを備えたモジュールレベルおよびグローバルエラーハンドラー、デバッグ用の詳細な実行ログ、ロールバック機能。

<strong>データストアと変数:</strong>実行間の永続ストレージ、複雑なステートフルワークフローのための変数管理。

<strong>テンプレートとアカデミー:</strong>数百の事前構築されたシナリオテンプレート、初心者から上級認定までの構造化された学習パス。

<strong>セキュリティとチーム管理:</strong>詳細なユーザー権限、認証付きの安全なWebhookエンドポイント、監査ログ、エンタープライズ展開用のSSO。

<strong>Make AI:</strong>インテリジェントなデータ抽出、スマートルーティング決定、自動最適化のための最近導入されたAI搭載機能。

## 一般的なユースケース

### eコマース自動化

<strong>注文処理:</strong>新しいShopify注文トリガー → Google Sheetsに追加 → 確認メール送信 → Slackチャンネルに通知 → 在庫更新

<strong>国際注文:</strong>国別に注文をフィルタリング → 適切なフルフィルメントセンターにルーティング → ローカライズされた配送ルールを適用 → 税関書類を生成

<strong>在庫管理:</strong>在庫レベルを監視 → しきい値を下回ったら自動再注文 → サプライヤーに通知 → 購買承認ワークフロー

### マーケティングオートメーション

<strong>リード管理:</strong>Facebook Lead Ads送信 → CRM(HubSpot、Salesforce)に追加 → メールシーケンス(Mailchimp)に登録 → 営業担当者を割り当て → フォローアップをスケジュール

<strong>キャンペーン追跡:</strong>複数の広告プラットフォームからメトリクスを集約 → レポートダッシュボードに統合 → ROIを計算 → 週次サマリーを生成 → ステークホルダーに配布

<strong>ウェビナー自動化:</strong>登録 → Google Sheetsに追加 → 確認送信 → リマインダーシリーズ → ウェビナー後のアンケート → フォローアップナーチャーシーケンス

### カスタマーサービス

<strong>チケットルーティング:</strong>ウェブサイトチャットまたはメール → コンテンツを解析 → キーワード/緊急度で分類 → 適切なチームにルーティング → Zendeskでチケット作成 → 割り当てられたエージェントに通知

<strong>満足度調査:</strong>ケースクローズトリガー → 24時間待機 → 満足度調査送信 → 回答を集約 → ネガティブフィードバックにフラグ → 問題をエスカレーション

<strong>ナレッジベース同期:</strong>ナレッジシステムで更新 → 影響を受ける顧客を特定 → 通知送信 → チャットボットトレーニングデータを更新 → 変更をログ記録

### 財務業務

<strong>トランザクション処理:</strong>Stripe/PayPalトランザクション → タイプ別に分類 → 会計スプレッドシートに追加 → 大額にフラグ → 請求書生成 → クラウドストレージにアーカイブ

<strong>経費管理:</strong>領収書アップロード → OCR抽出 → ポリシーに対して検証 → 承認のためにルーティング → 会計システムに記録 → 従業員に払い戻し

<strong>レポート作成:</strong>複数のソースからデータを集約 → メトリクスを計算 → フォーマットされたレポートを生成 → メール/Slack経由で配布 → 履歴データをアーカイブ

### 人事と採用

<strong>応募者追跡:</strong>求人サイト応募 → Airtableに追加 → 履歴書を解析 → 資格をスクリーニング → 面接をスケジュール → 候補者とのコミュニケーション送信 → ステータスを更新

<strong>オンボーディング:</strong>新規採用トリガー → アカウント作成(メール、Slack、プロジェクト管理) → 機器を割り当て → トレーニングに登録 → ミーティングをスケジュール → ウェルカム資料を送信

<strong>勤怠管理:</strong>タイムシート送信 → 時間を検証 → 承認のためにルーティング → 給与に同期 → 残業を計算 → レポートを生成

## 実装ガイド

### はじめに

<strong>アカウント作成:</strong>Make.comで月間1,000オペレーションをサポートする無料ティアでサインアップします。

<strong>開始点を選択:</strong>要件に基づいてテンプレートライブラリを閲覧するか、ゼロから新しいシナリオを作成します。

<strong>トリガーを設定:</strong>トリガーアプリとイベント(例:「新しいGoogleフォーム回答を監視」)を選択し、接続を認証し、トリガーパラメータを指定します。

<strong>アクションを追加:</strong>各ステップのモジュール(メール送信、データベース更新、API呼び出し)をドラッグし、各パラメータと認証を設定します。

<strong>データをマッピング:</strong>ビジュアルデータマッパーを使用して、前のモジュールの出力を後続のモジュールの入力に接続し、利用可能なフィールドを表示します。

<strong>ロジックを追加:</strong>不要なデータフローを防ぐフィルター、条件に基づいて分岐するルーター、配列を要素ごとに処理するイテレーターを挿入します。

<strong>エラー処理を設定:</strong>再試行または代替アクションのためのモジュールレベルのエラーハンドラー、シナリオ全体の障害のためのグローバルハンドラーを設定します。

<strong>徹底的にテスト:</strong>サンプルデータを使用して「一度実行」を実行し、エラーを特定する詳細なログを確認し、データ変換が期待される結果を生成することを検証します。

<strong>スケジュールと有効化:</strong>スケジューリング間隔またはWebhookトリガーを設定し、本番使用のためにシナリオを有効にします。

<strong>監視と最適化:</strong>実行ログを定期的に確認し、データマッピングを改善し、パフォーマンスと信頼性のために最適化します。

## ベストプラクティス

<strong>シンプルに始める:</strong>複雑なワークフローにスケールする前に小規模な自動化を構築してテストし、圧倒的な複雑さを回避します。

<strong>テンプレートを使用:</strong>事前構築されたテンプレートを開始点として活用し、特定のニーズに適応させて開発時間を短縮します。

<strong>わかりやすく命名:</strong>モジュール、変数、ルーターに明確なラベルを付け、チームメンバーによるコラボレーションとメンテナンスを可能にします。

<strong>早期にフィルターを活用:</strong>不要なデータがシナリオを通じて伝播するのを防ぎ、不要なオペレーションとコストを削減します。

<strong>実行ログを監視:</strong>ログを定期的に確認し、障害、ボトルネック、最適化の機会を特定します。

<strong>複雑なロジックを文書化:</strong>ビジネスルール、決定ポイント、データ変換を説明するメモを追加し、将来の参照用にします。

<strong>Webhookを保護:</strong>認証トークン、IPホワイトリスト、HTTPS接続を使用して機密データを保護し、不正アクセスを防ぎます。

<strong>実データでテスト:</strong>実際のアカウントと代表的なデータセットを使用して、自動化がエッジケースと本番条件を処理することを確認します。

<strong>エラーを積極的に処理:</strong>予測可能な障害のためにエラーハンドラーを設定し、注意が必要な重大なエラーの通知を設定します。

<strong>パフォーマンスを最適化:</strong>大規模なワークフローをサブシナリオに分割し、永続状態のためにデータストアを使用し、適切な場合はバッチ操作を使用します。

## 価格

<strong>無料ティア:</strong>すべてのコア機能と2,500以上のアプリへのアクセスを備えた月間1,000オペレーション。

<strong>コアプラン:</strong>月額$10.59から、オペレーション制限の増加と高度な機能。

<strong>プロプラン:</strong>より高いボリューム、優先サポート、高度なチームコラボレーション。

<strong>エンタープライズ:</strong>カスタム制限、専用サポート、SSO、監査ログ、SLA保証。

<strong>オペレーションカウント:</strong>各モジュール実行が1オペレーションとしてカウントされます—多くのモジュールを持つ複雑なシナリオはオペレーションを迅速に消費します。

## よくある質問

<strong>MakeはIntegromatと同じですか?</strong>はい、MakeはIntegromatのリブランド名です。すべての機能とワークフローは完全に互換性があります。

<strong>コーディングスキルは必要ですか?</strong>いいえ。ほとんどのシナリオはコーディングを必要としません。上級ユーザーはカスタム関数、HTTPモジュール、正規表現を活用できます。

<strong>価格設定はどのように機能しますか?</strong>オペレーション(各モジュール実行)に基づきます。無料ティアは月間1,000オペレーションを提供し、有料プランは上方にスケールします。

<strong>主な制限は何ですか?</strong>複雑な機能の学習曲線、オペレーションベースの価格設定が増加する可能性、プランによるサポート応答時間の変動。

<strong>どの業界がMakeを使用していますか?</strong>eコマース、マーケティングエージェンシー、SaaS企業、金融、人事、カスタマーサポート、IT運用、教育。

<strong>Makeは大量のデータを処理できますか?</strong>はい、適切なシナリオ設計で可能です。高ボリュームワークフローにはバッチ処理、データストア、最適化技術を使用します。

<strong>Makeはどの程度安全ですか?</strong>Webhook認証、暗号化接続、詳細な権限、監査ログ、エンタープライズプラン向けのSOC 2コンプライアンスを提供します。
