## ハンバーガーメニューとは?

**ハンバーガーメニュー**は、グラフィカルユーザーインターフェース(GUI)要素の一つで、最も一般的には3本の水平線を重ねた形(「ハンバーガーアイコン」)で表現されます。クリックまたはタップすると、ナビゲーションオプション、設定、または追加機能を含む隠されたメニューが表示されます。

**AIチャットボットや自動化プラットフォーム**では、ハンバーガーメニューはチャットウィンドウ内に埋め込まれることが多く、アカウント設定、チャット履歴、ヘルプ、ボット間の切り替えなどの常設オプションへのアクセスを提供し、主要な会話インターフェースを圧迫しません。

**なぜこの名前?** このアイコンの3本の線が、様式化されたハンバーガー(2つのバンズの間にパティ)を視覚的に連想させるためです。この比喩は現在、隠しナビゲーションを表すほぼ普遍的なものとなっています。

*参考文献: [Interaction Design Foundation: Hamburger Menu Guide](https://www.interaction-design.org/literature/article/hamburger-menu-ux)*

## 歴史的起源と進化

- **初登場:**  
  ハンバーガーアイコンは、1981年にデザイナーのノーム・コックスがXerox Starパーソナルワークステーション用に作成し、オプションのリストを表現するために使用されました。限られたピクセル空間(16x16pxまたは13x13px)のため、意図的にシンプルで記憶に残るデザインでした。[コックスのインタビューを読む](https://medium.com/readme-mic/a-conversation-with-norm-cox-creator-of-the-hamburger-menu-c913daea5f9e)、[Interaction Design Foundationの歴史](https://www.interaction-design.org/literature/article/hamburger-menu-ux)を参照。

- **初期の採用:**  
  Xerox StarとWindows 1.0(1985年)での使用後、ハンバーガーメニューは数十年間、主流のインターフェースから姿を消しました。

- **モバイルでの復活:**  
  2000年代後半のモバイルアプリの爆発的普及により、ハンバーガーメニューが復活しました。2008年のTwitterのモバイルアプリとAppleのiOSが画面スペースを最大化するために使用し、その後Facebook、Gmail、Amazonなどのアプリでも採用されました。

- **現在の状況:**  
  ハンバーガーメニューは、検索の虫眼鏡や設定の歯車アイコンと同じくらい認知されています。モバイルおよびレスポンシブWebナビゲーションの標準的なソリューションです。

*ビジュアルタイムライン: [Justinmindの図解された歴史](https://www.justinmind.com/ui-design/hamburger-menu)、[NNGの分析](https://www.nngroup.com/articles/hamburger-menu-icon-recognizability/)。*

## ハンバーガーメニューの使用方法と場所

### 典型的な使用例

- **モバイルアプリとレスポンシブWebサイト:**  
  ハンバーガーメニューは、小さな画面で重要度の低いナビゲーションを隠し、視覚的なスペースを確保するために使用されます。

- **AIチャットボットと自動化:**  
  チャットインターフェースでは、ハンバーガーメニューにユーザー設定、チャット履歴、ボット切り替え、ヘルプドキュメントなどが含まれることがよくあります。

- **デスクトップアプリケーション:**  
  あまり一般的ではありませんが、二次的なツールや設定に使用されることがあります。

- **機能豊富なWebアプリ:**  
  多くの二次的なオプション(例:Uberの支払い設定、Gmailのフォルダ構造)を持つアプリは、コアワークフローを整理するためにハンバーガーメニューを使用します。

*出典: [Interaction Design Foundation](https://www.interaction-design.org/literature/article/hamburger-menu-ux)*

### AIチャットボットと自動化での例

- **チャットボット設定:**  
  IntercomやDriftなどのツールは、チャットウィジェット内のハンバーガーメニューにユーザー設定、通知、リソースを配置します。

- **常設オプション:**  
  言語切り替え、プライバシー管理、フィードバックフォームなどは、ハンバーガーメニューからアクセスされることが多いです。

- **マルチボットナビゲーション:**  
  エンタープライズダッシュボードでは、ユーザーはハンバーガーメニューを使用してボットやチャネル間を切り替えることができます。

## 主要なUXコンセプト:プログレッシブディスクロージャーとインタラクションコスト

### プログレッシブディスクロージャー

**プログレッシブディスクロージャー**は、最も関連性の高い情報のみをデフォルトで表示し、必要に応じて追加オプションを表示するデザイン原則です。ハンバーガーメニューは典型的な例で、主要なアクションは表示されたままで、二次的なオプションはオンデマンドで利用可能です。

- **利点:** 視覚的な混乱と認知的負荷を軽減し、特にモバイルで効果的です。
- **欠点:** 重要なオプションを隠すリスクがあり、発見可能性を低下させる可能性があります。

*詳細: [Interaction Design Foundation: Progressive Disclosure](https://www.interaction-design.org/literature/topics/progressive-disclosure)*

### インタラクションコスト

**インタラクションコスト**は、ユーザーが目標を達成するために必要な時間、労力、アクション数です。ハンバーガーメニューは、ユーザーが機能にアクセスするために追加のタップまたはクリックを実行する必要があるため、インタラクションコストを増加させます。

- **高いインタラクションコスト:** ユーザーのエンゲージメントが低下し、タスク完了が遅くなる可能性があります。
- **低いインタラクションコスト:** 効率性が向上しますが、視覚的な混乱が増加する可能性があります。

*参考文献: [NNG: Interaction Cost Definition](https://www.nngroup.com/articles/interaction-cost-definition/)*

## ハンバーガーメニューの長所と短所

### 長所

- **インターフェースを簡素化:**  
  ナビゲーションオプションを隠すことで、よりクリーンで混乱の少ないインターフェースを作成します。ミニマリストデザインやモバイルデザインで特に価値があります。
- **モバイルユーザビリティの向上:**  
  限られた画面スペースを最大化し、タップでアクションにアクセス可能にします。
- **多数のリンクに対応:**  
  ユーザーを圧倒することなく、複雑なナビゲーション構造を処理します。
- **プログレッシブディスクロージャーをサポート:**  
  二次的な機能をアクセス可能に保ちながら、邪魔にならないようにします。
- **普遍的なアイコン:**  
  ほとんどのユーザーに認識されており、信頼できるメンタルモデルとなっています。
- **一貫性を促進:**  
  複数のプラットフォーム間で統一されたナビゲーションアプローチを提供します。

*出典: [Interaction Design Foundation](https://www.interaction-design.org/literature/article/hamburger-menu-ux)*

### 短所

- **発見可能性の低下:**  
  隠されたナビゲーションは、重要なセクションや機能の可視性を低下させます。
- **ナビゲーションの遅延:**  
  追加のクリック/タップが必要で、迅速なアクセスを必要とするユーザーをイライラさせる可能性があります。
- **開いたときに圧倒される:**  
  大きなメニューは多くのオプションを提示し、ユーザーを圧倒する可能性があります。
- **一部のユーザーにとって馴染みがない:**  
  すべての人口統計がハンバーガーアイコンを認識するわけではありません。高齢者や技術に不慣れなユーザーは苦労する可能性があります。
- **常に適切とは限らない:**  
  シンプルなサイトや即座のアクションが必要なシナリオには不向きです。
- **アクセシビリティの障壁:**  
  適切なラベルとキーボードサポートで実装されていない場合、障害を持つユーザーの妨げになる可能性があります。

*参考文献: [NNG: Hamburger Menus Hurt UX Metrics](https://www.nngroup.com/articles/hamburger-menus/)、[UserWay: UX & Accessibility Best Practices](https://userway.org/blog/website-navigation/)、[HubSpot: Hamburger Button Pros & Cons](https://blog.hubspot.com/website/hamburger-button)*

## デザイン原則とベストプラクティス

### ビジュアルデザインガイドライン

- 古典的な3本線のアイコンを使用—装飾なし、等しい長さ。
- 一貫した配置(ほとんどの言語では左上、RTLまたは親指に優しいレイアウトでは右上)。
- 不要な境界線や装飾を避ける。
- 馴染みのない視聴者や独特なコンテキストには「メニュー」ラベルを追加。
- クリック可能性を確保:ホバー状態とカーソル変更を使用。
- タップしやすい十分なサイズとパディングを提供。
- アイコンと背景の高いコントラスト。
- 微妙なアニメーション(例:開いたときに「X」に変形)でインタラクティブ性を強化。

*ガイダンス: [NNG: Hamburger Icon Best Practices](https://www.nngroup.com/articles/hamburger-menu-icon-recognizability/)、[Justinmind: Hamburger Menu Design Guide](https://www.justinmind.com/ui-design/hamburger-menu)*

### UXガイドライン

- コア機能を隠さない。二次的または頻度の低いアクションに使用。
- ハイブリッドナビゲーションを検討—可視タブ/バーと、あまり使用されないアイテム用のハンバーガーを組み合わせる。
- 見落とされたリンクや機能について分析を監視。
- アイコンの認識とアクセシビリティをユーザーテスト。
- アイコンの混乱を避ける—類似のアイコンを近くに配置しない。
- 高速でスムーズなメニューの開閉を確保。

### アクセシビリティの考慮事項

- セマンティックHTMLを使用:`<button>`要素をメニュートリガーに。
- スクリーンリーダー用に`aria-label="Menu"`または類似のものを追加。
- キーボードでメニューにアクセス可能にする(Tabで開く、矢印キーでナビゲーション)。
- すべてのインタラクティブ要素に可視フォーカス状態を提供。
- 十分な色のコントラストと大きなタッチターゲットを確保。

*アクセシビリティ: [UXDesign.cc: Create an Accessible Hamburger Menu](https://uxdesign.cc/create-an-accessible-hamburger-menu-869b0301cfd7)、[UserWay: Accessibility Navigation Best Practices](https://userway.org/blog/website-navigation/)*

## 実世界の例

### Webおよびモバイルアプリケーション

- **Gmail(Android/iOS):** 左上のハンバーガーメニューでフォルダとアカウントオプションを表示。  
  [Gmail on Google Play](https://play.google.com/store/apps/details?id=com.google.android.gm)
- **Uber:** ハンバーガーメニューは旅行履歴、領収書、設定に使用されます。
- **Amazonモバイル:** ハンバーガーメニューで二次的なナビゲーションと設定にアクセス。
- **Pipcorn:** モバイルハンバーガーメニューが折りたたみナビゲーションパネルをトリガー。  
  [Pipsnacks.com](https://www.pipsnacks.com/)

### AIチャットボットインターフェース

- **エンタープライズチャットボットウィジェット:** チャットウィンドウ内のハンバーガーメニューでプロフィール、通知設定、チャット履歴、サポートにアクセス。
- **カスタマーサポートボット:** メニューにはナレッジ記事とエスカレーションオプションへのリンクが含まれます。
- **マルチボットプラットフォーム:** メニューでボットやチャネル間の切り替えが可能。

## ハンバーガーメニューの代替案

ハンバーガーメニューの代わりに、または併用して検討すべきナビゲーションパターン:

- **タブバー:** 可視のナビゲーションオプションの行。3〜5つのコアセクションに最適(Instagram、Facebook)。
- **ボトムナビゲーションバー:** モバイルフレンドリーで、主要なアクションを親指の届く範囲に保ちます。
- **可視サイドバー:** デスクトップまたは大画面用。常に表示されるサイドメニュー。
- **ドロップダウンメニュー:** トップバーからホバーまたはクリックでメニューをトリガー(デスクトップ)。
- **フローティングアクションボタン(FAB):** 単一の主要なアクションに目立つ。

### 比較とどのパターンを使用すべきか

| パターン | 発見可能性 | スペース効率 | 適している用途 |
| -------------- | -------------- | --------------- | -------------------- |
| ハンバーガーメニュー | 低 | 高 | モバイル、二次ナビ |
| タブバー | 高 | 中 | モバイル、コアナビ |
| ボトムバー | 高 | 中 | モバイル、頻繁なナビ |
| サイドバー | 高 | 低 | デスクトップ、大画面 |

ユーザーのニーズと提供するオプションの重要性に合ったモデルを選択してください。

*比較: [Justinmind: Navigation Patterns](https://www.justinmind.com/blog/navigation-design-almost-everything-you-need-to-know/)*

## ハンバーガーメニューを使用または回避すべき場合

**ハンバーガーメニューを使用すべき場合:**
- スペースが限られている(モバイルファーストデザイン)。
- ナビゲーションオプションが可視インターフェーススペースを超える。
- オプションが二次的または重要でない。
- デバイス間の一貫性が重要。

**ハンバーガーメニューを回避または再考すべき場合:**
- 重要な機能が隠される。
- 分析で隠されたアイテムへのエンゲージメントが低い。
- ターゲットユーザーがアイコンに馴染みがない。
- エクスペリエンスがすでにインタラクションが多い。
- インターフェースに可視ナビゲーションのスペースがある。

*詳細: [Interaction Design Foundation](https://www.interaction-design.org/literature/article/hamburger-menu-ux)*

## 実装ノートと実用的なヒント

### HTML/CSSでのシンプルなハンバーガーメニュー

**静的アイコンの例:**

```html
<div class="hamburger-menu">
  <div></div>
  <div></div>
  <div></div>
</div>
```

```css
.hamburger-menu div {
  width: 35px;
  height: 5px;
  background-color: #333;
  margin: 6px 0;
}
```

**アクセシブルでインタラクティブな例:**

```html
<button aria-label="Open menu" class="hamburger-menu" tabindex="0">
  <span></span>
  <span></span>
  <span></span>
</button>
```

- セマンティクスとキーボードナビゲーションのために`<button>`を使用。
- スクリーンリーダー用に`aria-label`を追加。
- 十分なタッチエリアと可視フォーカス状態を確保。

*デモ: [CodePen Example](https://codepen.io/Danilo06/pen/PoNNvGm)、[UXDesign: Accessible Hamburger Menu](https://uxdesign.cc/create-an-accessible-hamburger-menu-869b0301cfd7)*

## まとめと推奨事項

- スペースが限られている場合、またはオプションが二次的な場合にのみハンバーガーメニューを使用。
- 可能な限り可視ナビゲーションをデフォルトに。
- 馴染みのあるアイコンパターンと配置に従う。
- 認識に疑問がある場合はメニューにラベルを付ける。
- 重要な機能を隠さない。主要なアクションは可視に保つ。
- 実際のユーザーでユーザビリティとアクセシビリティをテスト。
- キーボードとスクリーンリーダーのサポートを確保。
