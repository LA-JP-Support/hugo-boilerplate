## Dublin Core とは?

**Dublin Core は、デジタルリソース（ウェブページ、電子書籍、データセット）を統一的に記述するための 15 の基本要素を定める国際標準です。** 1995年にアイルランドのダブリンで開催されたメタデータのワークショップで生まれたため、この名称になりました。例えば、「タイトル」「作成者」「主題」「説明」「出版者」などの情報を構造化して記述します。この標準により、異なる図書館やデータベース間でリソース情報を相互利用できるようになります。[メタデータ](Dublin-Core.md)管理が重要になるデジタル時代に、極めて重要な仕組みです。

> **ひとことで言うと：** 「デジタルな本や文書について『タイトル』『作成者』『説明』などの情報を、すべての図書館が同じ形式で記録する方法」です。

**ポイントまとめ：**

- **何をするものか：** デジタルリソースの情報を標準的な形式で記述
- **なぜ必要か：** 異なるシステム間での情報交換が可能になる
- **誰が使うか：** 図書館員、データ管理者、デジタルアーカイブ運営者

## なぜ重要か

Dublin Core が重要な理由は三つあります。第一に、「相互運用性」を実現します。同じ形式で記述されていれば、異なる図書館やデータベース間で検索できます。第二に、シンプルで実装しやすいです。複雑なメタデータ標準と異なり、15 要素だけで基本的な記述ができます。第三に、国際的に認められた標準（ISO 15836）であり、長期的な信頼性があります。[セマンティックウェブ](Docker.md)の発展とともに、Dublin Core は RDF（リソース記述フレームワーク）にも統合されています。

## 仕組みをわかりやすく解説

Dublin Core の仕組みはシンプルです。まず、デジタルリソース（例：PDF ファイル）を記述する際に、「タイトル」「作成者」「作成日」などのメタデータを付与します。これらは XML や RDF フォーマットで保存されます。次に、このメタデータをシステム間で交換します。OAI-PMH（Open Archives Initiative Protocol for Metadata Harvesting）というプロトコルを使い、複数の図書館が自動的にメタデータを収集できます。

例えば、大学 A の電子リポジトリと大学 B の電子リポジトリが同じ Dublin Core 形式でデータを記述していれば、統合検索で両方のコンテンツが見つかります。

## 実際の活用シーン

**デジタルアーカイブ** 歴史的文書や古い写真をデジタル化し、Dublin Core で記述して、オンラインで公開します。

**大学の学位論文リポジトリ** 卒業生の論文を Dublin Core フォーマットで登録し、他大学と共有します。

**美術館の収蔵品データベース** 美術作品の情報（タイトル、作成者、製作年、材質）を Dublin Core で管理します。

## メリットと注意点

**メリット：** シンプルで学習が簡単です。費用がかからず（オープン標準）、多くのツールがサポートしています。異なる組織間でのデータ交換が容易です。

**注意点：** 15 個の基本要素では、複雑なリソースを詳細に記述できない場合があります。品質管理が重要ですが、大規模コレクションでは困難です。異なる組織が異なる方法で要素を解釈する可能性があります。

## よくある質問

**Q: 15 個すべての要素を使う必要がありますか?**
A: いいえ。リソースの性質に応じて、必要な要素だけ使用できます。最小限は「タイトル」程度です。

**Q: Dublin Core の拡張要素を使用できますか?**
A: はい。より詳細な記述が必要な場合、拡張要素（Qualified Dublin Core）を使用できます。

**Q: 他の図書館との相互運用性を確保するには?**
A: OAI-PMH などのハーベスティングプロトコルを使用し、メタデータを定期的に交換します。
