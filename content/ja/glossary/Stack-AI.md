## 定義
**Stack AI**は、高度な大規模言語モデル(LLM)を中心とした、AI駆動型ワークフローとエージェントの構築、テスト、デプロイを行うためのエンタープライズグレードのノーコード/ローコードプラットフォームです。Stack AIは、組織が分散したデータソースやビジネスシステムを接続し、複雑なプロセスを自動化し、AIアシスタントを大規模にデプロイすることを可能にします—すべて深いプログラミング知識を必要とせずに実現できます。

- **公式ウェブサイト:** [https://www.stack-ai.com/](https://www.stack-ai.com/)
- **詳細レビュー:** [Marketer Milk Stack AI Review](https://www.marketermilk.com/blog/stack-ai-review)
- **Voiceflow比較:** [Voiceflow Stack AI Review](https://www.voiceflow.com/blog/stack-ai)

## Stack AIとは?

Stack AIは、元MIT博士号取得者のAntoni RosinolとBernardo Aceitunoによって設立され、ビジネスデータと最先端のAIモデルを接続するという課題に対処するために作られました。Stack AIは、安全でコンプライアンスに準拠した、スケーラブルなAI駆動型オペレーションを必要とする企業をターゲットとしています。このプラットフォームは、モジュール式の自動化を構築し、LLMを統合し、複数ステップのビジネスワークフローをオーケストレーションするためのドラッグアンドドロップインターフェースを提供します。

### 主な特徴

- **ノーコード/ローコードビルダー:** ドラッグアンドドロップによるワークフロー作成、モジュール式ステップ、平易な英語による説明で自動化を構築。
- **LLM統合:** OpenAI、Anthropic Claude、Google Gemini、Cohereモデルのネイティブサポート。
- **セキュリティ/コンプライアンス:** SOC2、HIPAA、GDPR準拠([セキュリティページ](https://www.stack-ai.com/security)参照)。
- **統合エコシステム:** 100以上のSaaS、クラウド、データベース、APIコネクタ。
- **柔軟なデプロイメント:** クラウド、オンプレミス、VPC、ウェブ、チャット、API、Slack、Teams、埋め込みウィジェット。
- **エンタープライズフォーカス:** 規制産業や大規模自動化向けに設計され、きめ細かなアクセス制御を備えています。

## Stack AIの使用方法

### 主な使用パターン

Stack AIは、LLMとビジュアルワークフロービルダーを組み合わせることで、知識駆動型、反復的、または労働集約的なビジネスワークフローを効率化します:

- **AIエージェント作成:** モジュール式ビルダーを通じて、カスタムチャットボット、ドキュメント分析ツール、ナレッジアシスタント、「RAG」ボットを構築。
- **ドキュメント処理:** PDF、契約書、スプレッドシートなどから大規模にデータを抽出、分類、処理。
- **ワークフロー自動化:** ビジネスロジックをオーケストレーション—例:メール取り込み、CRMクエリ、レポート生成、Slackへの投稿。
- **データ統合:** SharePoint、Salesforce、Notion、Snowflake、Google Driveなどのプラットフォームから非構造化データを接続。
- **カスタムインターフェース:** ウェブウィジェット、フォーム、Slack/Teamsボット、またはAPIエンドポイントを通じてデプロイ。

**ワークフロー例:**  
Google DriveからPDFを読み込み → OCRで主要データを抽出 → ClaudeまたはGPT-4で分析 → Salesforceに要約を書き込み → Slackでチームに通知。

### Stack AIの利用者

- **金融・保険:** ドキュメントレビューとコンプライアンスの自動化([金融ソリューション](https://www.stack-ai.com/solutions/finance)および[保険ソリューション](https://www.stack-ai.com/solutions/insurance)参照)。
- **ヘルスケア:** 医師向けのHIPAA準拠コパイロットの構築、カルテ作成の自動化、EHR抽出([ヘルスケアソリューション](https://www.stack-ai.com/solutions/healthcare))。
- **教育:** AIチューター、ナレッジアシスタント、管理ボット([教育ソリューション](https://www.stack-ai.com/solutions/education))。
- **エージェンシー・マーケティング:** コンテンツQA、オンボーディングフロー、ナレッジボット。
- **IT・オペレーション:** バックオフィス自動化、RFP管理、データ抽出。
- **非技術系ユーザー:** ノーコードインターフェースにより、ビジネスアナリストやオペレーションマネージャーが自動化を構築可能。
- **技術リーダー:** 開発者向けのAPIと深い統合サポート。

**顧客事例:** HP、SmartAsset、Red Bull、IBM、MIT。

## Stack AIの主要機能

### 1. ノーコードワークフロービルダー

- **ビジュアルドラッグアンドドロップ:** モジュール式「ノード」(データソース、ロジック、AIモデル、出力)からワークフローを構成。
- **クイックスタート/カスタムモード:** テンプレートを使用するか、ゼロから設計。
- **会話型ビルダー:** 平易な英語で説明することでワークフローを構築。
- **高度なロジック:** 並列、条件分岐、複数ソース集約をサポート。
- **再利用可能なコンポーネント:** ワークフローテンプレートを組織全体で保存・再利用。

**出典:**  
- [Voiceflow Stack AI Review](https://www.voiceflow.com/blog/stack-ai)

### 2. 統合エコシステム

SaaS、クラウドストレージ、データベース、コミュニケーションプラットフォーム、APIとの100以上の統合:

- **データ:** Google Drive、OneDrive、Snowflake、SharePoint、Notion、MongoDB、Airtable、AWS S3、BigQueryなど。
- **ビジネスアプリ:** Salesforce、HubSpot、Zendesk、GitHub。
- **メッセージング:** Slack、Microsoft Teams、WhatsApp、SMS。
- **AIプロバイダー:** OpenAI、Anthropic、Google、Cohere。
- **カスタムAPI:** 任意のRESTfulサービスに接続。マネージドカスタムプロセス(MCP)ノードにより高度な統合が可能。

**統合設定:** OAuth認証、ワークフロー間で再利用可能。

- [Stack AIドキュメント – 統合](https://docs.stack-ai.com/integrations)

### 3. マルチチャネルインターフェース

- **ウェブチャットボット:** ウェブサイトや社内ポータルに埋め込み可能。
- **フォーム:** AI駆動型フィードバック付きの構造化データ収集。
- **バッチ処理:** ドキュメントを一括分析。
- **Slack/Teamsボット:** コラボレーションツールにAIアシスタントを導入。
- **APIエンドポイント:** カスタムアプリケーションにワークフローを埋め込み。
- **カスタムUIブランディング:** 企業ブランディングでチャットボットやフォームをカスタマイズ。
- **きめ細かなアクセス制御:** 各インターフェースの使用・変更権限を管理。

### 4. 高度なAI機能

- **検索拡張生成(RAG):** LLMを独自データに接続し、引用付きのコンテキスト対応回答を提供。
- **データ抽出/分類:** スキャンドキュメント用のOCR、構造化データ抽出。
- **プロンプトエンジニアリング:** LLM指示の調整、フィードバック収集、エージェント動作の反復改善。
- **カスタムLLM選択:** 異なるワークフローステップに特定のモデル(例:GPT-4、Claude 3)を割り当て。
- **ナレッジベース検索:** セマンティック検索のためのベクトルデータベースサポート。

### 5. セキュリティ、コンプライアンス、アクセス制御

- **コンプライアンス:** SOC2、HIPAA、GDPR認証—規制産業に適合([セキュリティ詳細](https://www.stack-ai.com/security))。
- **デプロイメント:** クラウド、オンプレミス、または専用インフラ用のVPC。
- **SSO:** エンタープライズIDプロバイダーと統合。
- **ロールベース権限:** きめ細かなアクセス、コンプライアンス用の監査証跡。

## ユースケース例

### 1. 金融ドキュメントレビュー

- **課題:** 大量のローンまたはクレジット申請処理。
- **ワークフロー:** ドキュメント取り込み → OCRとLLMによる主要条件の抽出 → データベースとのクロスリファレンス → レポート生成 → Slackで通知。
- **成果:** レビュー時間が数時間から数分に短縮、手動エラーの削減。

### 2. ヘルスケア患者コパイロット

- **課題:** 医師の管理時間を削減。
- **ワークフロー:** EHRと統合 → 履歴要約、SOAPノート生成 → Teamsで安全なチャットボットをデプロイ。
- **成果:** ドキュメント作成の改善、患者ケアにより多くの時間。

### 3. AIナレッジアシスタント

- **課題:** 従業員が社内ドキュメントから即座に正確な回答を必要とする。
- **ワークフロー:** ナレッジベースを読み込み → RAGワークフローで関連情報を取得 → LLMが引用付き回答を生成 → ウェブ/Slackボットとしてデプロイ。
- **成果:** 生産性向上、ヘルプデスクチケットの削減。

### 4. 自動RFP管理

- **課題:** 退屈で反復的なRFP/アンケート回答。
- **ワークフロー:** RFPを取り込み → LLM/テンプレートで回答草案を作成 → レビューにルーティング → 提出と進捗追跡。
- **成果:** より迅速で一貫性のあるRFP対応。

### 5. マーケティングコンテンツQA

- **課題:** マーケティングコピーの量と品質のレビュー。
- **ワークフロー:** 草案を取得 → LLMで文法、コンプライアンスをチェック → フィードバックを要約 → マネージャーに通知。
- **成果:** より迅速で高品質なレビュー、リスク低減。

## 技術アーキテクチャ

Stack AIの技術基盤はモジュール式でクラウドネイティブであり、柔軟性とスケーラビリティを重視しています:

- **ノードベースワークフローエンジン:** 各ステップはモジュール式ノード(データ、ロジック、AI、出力)。
- **コネクタエコシステム:** SaaS、データベース、API用の100以上のコネクタ。
- **モデルオーケストレーション:** ワークフローステップごとにLLMを割り当て/管理;プロンプト/コンテキスト注入をサポート。
- **デプロイメント:** SaaS、VPC、またはオンプレミス。
- **モニタリング/ログ:** 詳細な実行履歴、エラー追跡、分析。
- **ベクトルデータベース:** セマンティック検索とRAGを強化。
- **機械学習/NLP:** 抽出、分類、要約のコア技術。
- **アクセス/セキュリティモジュール:** コンプライアンスと保護を強制。

## 制限事項と考慮事項

- **価格:** プロトタイピング用の無料ティア(月500回実行);商用デプロイメントにはカスタムエンタープライズプランが必要([価格FAQ](https://docs.stack-ai.com/pricing))。
- **学習曲線:** 高度なワークフローには、LLM、データパイプライン、統合ロジックの理解が必要。
- **オンボーディング:** 複雑なケースに対するステップバイステップのオンボーディングが限定的との指摘あり。
- **統合カバレッジ:** すべてのSaaS/分析ツールが標準でサポートされているわけではない(例:2025年時点でGoogle Analyticsが欠落)。
- **サポート:** 無料ユーザー向けコミュニティ、エンタープライズ向け専任エンジニア。

## よくある質問

**開発者でなくてもStack AIを使用できますか?**  
はい。Stack AIのノーコードインターフェースとテンプレートにより、ビジネスユーザーでもアクセス可能です。([公式ドキュメント](https://docs.stack-ai.com/stack-ai/)参照)

**Stack AIはどのようなAIモデルをサポートしていますか?**  
OpenAI(GPT-4、GPT-3.5)、Anthropic(Claude 3)、Google Gemini、Cohereなど。

**Stack AIはデータプライバシーをどのように扱いますか?**  
SOC2、HIPAA、GDPR準拠;オンプレミス/VPCデプロイメントとロールベースアクセスをサポート。

**Stack AIは中小企業に適していますか?**  
主にエンタープライズ向け。プロトタイプ用の無料ティアあり;中小企業は代替プラットフォームを好む場合があります。

**典型的なStack AIワークフローとは?**  
入力(ファイル、メール、API) → データ抽出(OCR、LLM) → 分析 → 出力(CRM、Slack、レポート)。
